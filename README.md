# 音声メモ → アイゼンハワー・マトリクス

玉井個人用のメモアプリ。思いついたことを音声で話す → 文字起こし → 4象限へ振り分ける。
単一 `index.html` ＋ localStorage ＋ GitHub Pages。既存の「催事 売上報告メーカー」と同じ作り。

## 使い方

1. 🎤 話す（またはテキスト欄に直接入力）
2. 一問一答で Q1 重要度 → Q2 緊急度 に答える
3. 自動で象限が決まる。違えば1タップで移動できる

## 象限

| | 緊急 | 緊急でない |
|---|---|---|
| **重要** | Ⅰ 必須（橙）締切のある仕事・重要な会議 | Ⅱ 生産性とバランス（緑）人間関係作り・計画や準備 |
| **重要でない** | Ⅲ 錯覚（青）重要でない電話・妨害や邪魔 | Ⅳ 無駄（茶）暇つぶし・意味のない行動 |

Ⅱ が主役の象限。期限が **3日前** になったメモは Ⅱ → Ⅰ へ自動昇格する（うっかり防止）。

## 判定ルール

- **重要** ＝ Q1 で「直結する」を選んだものだけ
- **緊急** ＝ Q2 で「今日〜数日」を選んだものだけ。「今週〜今月」は緊急でない側に置く

「今週〜今月」を緊急側に倒すとほぼ全件が Ⅰ・Ⅲ に溜まり、Ⅱ が機能しなくなるため。
取りこぼしは期限3日前の自動昇格で拾う。※この方針は玉井の最終判断待ち。

## できること / できないこと（盛らない）

**できる**
- 音声または手入力 → 一問一答 → 4象限へ振り分け
- 音声から期限・相手を自動抽出してタイトル化（例「来月までに」→ 翌月末）
- 期限3日前で Ⅱ → Ⅰ 自動昇格
- オフラインでも画面は開く（Service Worker がシェルをキャッシュ）

**まだできない（次の工程）**
- 通知・バイブの実際の発火。現状は設定を記録しているだけ
- `.ics` 書き出し、Notion/LINE 貼り付け用テキスト

**原理的にできない（OS制約）**
- アプリを閉じている間の、マナーモードを貫通する爆音アラーム
- iOS ではバックグラウンドの予約通知が不安定
- 音声認識はオフライン不可・精度は端末依存。iOS Safari は対応にムラがあるため、
  非対応時はテキスト入力へ自動フォールバックする

確実に届かせたい用件は、`.ics` 実装後に Google カレンダー / iOS 純正リマインダーへ逃がすのが現実解。

## データ

端末内 `localStorage` のみ。キーは `gokaku_memo_matrix_v1`。サーバへは一切送らない。
ブラウザのデータを消すとメモも消える。

## ローカルで動かす

マイクと Service Worker は `https://` か `http://localhost` でしか動かない
（`file://` で開くとマイクが拒否される）。

```
node serve-local.js
```

→ http://localhost:8787/

`serve-local.js` は確認用。GitHub Pages では不要。

## 公開設定

`<meta name="robots" content="noindex,nofollow,noarchive">` ＋ `robots.txt` で検索避け。
ただし noindex は検索結果に出さないだけで、URL を知っていれば誰でも開ける。
